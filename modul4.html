Tags: Flexible KategorienTags: Flexible Categories

Tags sind wie Sticker auf Notizen — eine Notiz kann viele Tags haben, anders als bei Ordnern. Tags are like stickers on notes — a note can have many tags, unlike folders.

📚 Die Cognitive Load Theory zeigt: Je weniger mentale Energie für Navigation und Suche nötig ist, desto mehr bleibt für das eigentliche Denken. Tags und Suche reduzieren die kognitive Belastung gegenüber manueller Ordner-Navigation. (Paas & van Merriënboer, 2020) 📚 Cognitive Load Theory shows: The less mental energy spent on navigation and search, the more remains for actual thinking. Tags and search reduce cognitive load compared to manual folder navigation. (Paas & van Merriënboer, 2020)

Wann Tags, wann Links? When tags, when links?

  • Tags für Kategorien/Status: #projekt, #erledigt, #idee, #dringend Tags for categories/status: #project, #done, #idea, #urgent
  • Links für inhaltliche Verbindungen: [[Max Müller]], [[Projekt Alpha]] Links for content connections: [[Max Müller]], [[Project Alpha]]
💡 Faustregel: Wenn du darüber schreiben würdest → Link. Wenn du danach filtern willst → Tag. 💡 Rule of thumb: If you'd write about it → Link. If you'd filter by it → Tag.

Suche: Schneller als jeder OrdnerSearch: Faster Than Any Folder

Drücke Ctrl/Cmd + Shift + F für die globale Suche. Obsidian durchsucht den kompletten Text aller Notizen in Millisekunden. Press Ctrl/Cmd + Shift + F for global search. Obsidian searches the full text of all notes in milliseconds.

Properties: Strukturierte MetadatenProperties: Structured Metadata

❌ Ohne Properties❌ Without Properties

Du suchst: "Alle aktiven Projekt-Notizen"
Du findest: 47 Notizen. Welche sind aktuell? 🤷 You search: "All active project notes"
You find: 47 notes. Which are current? 🤷

✅ Mit Properties✅ With Properties

Du suchst: [status:aktiv]
Du findest: 3 Notizen, sortiert nach Datum. ✅ You search: [status:active]
You find: 3 notes, sorted by date. ✅

Properties (seit Obsidian 1.4) sind strukturierte Felder am Anfang einer Notiz. Sie sehen so aus: Properties (since Obsidian 1.4) are structured fields at the top of a note. They look like this:

---
title: Projektplan Alpha
date: 2026-03-23
status: aktiv
tags: [projekt, alpha, Q2]
---

# Projektplan Alpha
...

Properties helfen beim Filtern und Sortieren. Du kannst in der Suche danach filtern: [status:aktiv]. Properties help with filtering and sorting. You can filter by them in search: [status:active].

Bookmarks: SchnellzugriffBookmarks: Quick Access

Für Notizen die du ständig brauchst: Rechtsklick → "Bookmark". Sie erscheinen im Bookmarks-Tab in der linken Sidebar. For notes you need constantly: Right-click → "Bookmark". They appear in the Bookmarks tab in the left sidebar.

Gute Kandidaten für Bookmarks: Good candidates for bookmarks:

  • Dein Dashboard / Home-NotizYour dashboard / home note
  • Aktuelle Projekt-NotizCurrent project note
  • Oft genutzte TemplatesFrequently used templates

🏋️ Challenge🏋️ Challenge

  1. Füge Tags zu 5 deiner bestehenden Notizen hinzu (z.B. #idee, #learning, #projekt) Add tags to 5 of your existing notes (e.g., #idea, #learning, #project)

  3. Füge Properties zu einer Notiz hinzu (klicke auf das Properties-Icon oben rechts oder schreibe ---) Add Properties to a note (click the properties icon top right or write ---)
  4. Bookmarke deine wichtigste Notiz Bookmark your most important note
